## 3. Data Security

### 3.1 Security Measures

We implement industry-standard security measures:

- **Encryption in Transit**: All data transmitted uses TLS 1.3 encryption
- **Encryption at Rest**: Data stored encrypted using AES-256 (via Supabase)
- **Secure Password Storage**: Passwords hashed using bcrypt/SHA-256
- **Authentication**: Email-based authentication with secure session management
- **Rate Limiting**: IP-based rate limiting to prevent abuse
- **Row Level Security**: Database access controls (RLS policies)

### 3.2 Infrastructure Security

- **Secure Hosting**: Deployed on Vercel with HTTPS enforced
- **Security Headers**: Content Security Policy (CSP), HSTS, X-Frame-Options
- **Regular Audits**: Dependency scanning and security assessments
- **Error Monitoring**: Sentry integration for real-time error tracking
- **Access Controls**: Strict access controls for our team

### 3.3 Payment Security

- **PCI-DSS Compliance**: PayMongo processes payments securely
- **No Card Storage**: We never store full credit card numbers
- **Secure Tokenization**: Payment tokens handled by PayMongo
- **Encrypted Transmission**: All payment data encrypted

### 3.4 What We Cannot Guarantee

While we take reasonable precautions:

- We cannot guarantee absolute security of transmitted information
- We protect against data breaches but cannot prevent all unauthorized access
- You are responsible for keeping your password secure
